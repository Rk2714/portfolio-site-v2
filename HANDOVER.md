## 3. リデザイン要件（次のエージェントが実施すべき内容）

### 3.1 絶対に避けること（NOリスト）
- [ ] **Interフォント**の使用（`font-family`から削除必須）
- [ ] 3カラムカードグリッドを連続して使用（最大1セクションのみ）
- [ ] フルスクリーン背景写真 + 中央揃え + 2CTAボタンのヒーロー
- [ ] グラデーションボタン（ナビゲーション「無料相談」ボタンも修正対象）
- [ ] カードのbox-shadow（**borderで区切る**）
- [ ] 絵文字アイコン
- [ ] **「RK」などのフェイクプロフィール画像**（写真がない場合は写真エリアごと削除）
- [ ] プログレスバーでのスキル表示
- [ ] 交互に変わるセクション背景色（#f5f5f5 / whiteパターン）

### 3.2 デザイン方向性
**テーマ**: 「医療現場の温かみ × テクノロジーの信頼感 × 沖縄の風」

**配色**:
- メイン: 白背景（#FFFFFF）
- アクセント: 1色のみ（青 #3B82F6 を維持可だが**グラデーション禁止**）
- テキスト: 濃い紺（#1e293b）
- 濃紺セクション: **フッターのみ**に絞る

**フォント**:
- **Interを完全に削除**
- `font-family: "Noto Sans JP", "Hiragino Sans", "Yu Gothic", sans-serif;`
- 見出し: Bold（700）、letter-spacing: -0.02em、line-height: 1.1-1.3
- 本文: line-height: 1.75
- ヒーロー見出し: 極大（text-5xl〜7xl）

### 3.3 セクション別レイアウト指示

#### ① ヒーロー（全面書き換え）
**現状の問題**: フルスクリーン背景+ダークオーバーレイ+中央揃え+2ボタン（AI SaaSテンプレ最頻出パターン）

**推奨案**:
```
- 背景: 白または okinawa-sea.jpg（沖縄の海）
- 左寄せレイアウト
- 左側60%: 大きな見出し（text-6xl〜7xl）
  「13年の看護経験を活かし、
   医療現場にAIの風を」
- 右側40%: 画像（hero-medical.jpg または okinawa-sea.jpg）
- CTA: 1つのみ。「まずはあなたの現場を聞かせてください」
- ナビゲーションの「無料相談」ボタン: フラット単色（グラデーション禁止）
```

#### ② 課題セクション（3カラム廃止）
```
- レイアウト: 縦並びの大きな数字付きリスト
- 左側: 大きな数字「01」「02」「03」（text-6xl、青、opacity-20）
- 右側: タイトル + 説明文
- 各項目の間に細いボーダー（border-t）
```

#### ③ サービスセクション（非対称レイアウト）
```
- 最重要サービス「AI導入コンサルティング」: 横長カード（1列）、画面幅いっぱい
  - 左側: 大きなアイコン
  - 右側: タイトル + 説明 + 箇条書き
  - 背景: 極淡いグレー（#FAFAFA）
- 残り2つ: その下に2カラムで小さく配置
```

#### ④ 実績セクション（優先順位を視覚化）
```
- 「ヌチマース号」（最重要実績）:
  - 画面幅の60%を占める特大カード
  - 背景に ambulance.jpg または okinawa-sea.jpg
  - 文字は白。オーバーレイ付き。
  - 「24時間対応」「沖縄発」を強調
- 残り4つ: シンプルなテキストリストまたは小さなカードで横に並べる
```

#### ⑤ プロフィールセクション（フェイク画像削除 ←最重要）
```
Option A（写真がある場合）:
- 左側: 実際の写真（縦長、角丸なしまたは大きな角丸）
- 右側: テキスト

Option B（写真がない場合）← 推奨:
- 「RK」エリアを完全に削除
- 全幅テキストレイアウト
- 上部に大きな名前「金城 竜弥」（text-5xl）
- その下に「KINJO RYUYA」（text-xl、tracking-widest）
- 経歴は縦のタイムライン形式
```

#### ⑥ スキルセクション（プログレスバー廃止）
```
- 3カラムは維持可だが、プログレスバーを削除
- 各スキル: 数字だけ大きく表示（text-4xl）
  例: "13年" "3施設" "50名+"
- 「言語スキル」部分はシンプルなテキストで十分
```

#### ⑦ FAQ・お問い合わせ

## 4. 追加予定のコンテンツ（別ページ・別スレッド対応）

**注意**: 以下のコンテンツは、別のAIエージェントとのやり取り（別スレッド）で対応予定です。現在のリデザイン時に取り込むか、Phase 2として後から追加するかは要相談。

### 4.1 メディア活動ページ

金城さんには**ラジオ出演・パーソナリティ活動・ゲスト紹介**の実績があります。

**想定ページ**: `/media` または `/radio`

**含めるべき内容**:
- ラジオ出演歴（出演した番組名、日時、テーマなど）
- パーソナリティとしての活動（レギュラー番組など）
- ゲスト紹介（紹介したゲストのプロフィール・写真・出演回など）
- 音声クリップやPodcastリンク（あれば）

**デザイン上の注意**:
- トップページのナビゲーションに「メディア」または「ラジオ」のリンクを追加する必要あり
- 実績セクションに「メディア出演」カテゴリを追加しても良いかもしれない
- **AI感を出さないための画像戦略**: 
  - ラジオブースの写真（Unsplashで `radio studio` `microphone` などで検索）
  - ゲストの写真（許可を得ている場合）
  - 音波・マイクのアイコン（Lucideの `Mic` `Radio` `Headphones` など）

**技術的対応**:
```
- 新規ページ: src/app/media/page.tsx
- または: src/app/radio/page.tsx
- ナビゲーション更新: src/components/Navbar.tsx（または相当コンポーネント）
```

## 8. 注意事項

1. **ビルド確認**: `npm run build` で静的エクスポート（`out/`ディレクトリ）を生成し、エラーがないことを確認してください。
2. **Vercelデプロイ**: GitHub連携済みなので、mainブランチにpushすれば自動デプロイされます。
3. **画像パス**: `public/images/` 以下の画像は `/images/xxx.jpg` で参照できます。
4. **実際の写真**: 金城さんの実際のプロフィール写真がない場合は、**無理に画像を置かず、テキストのみのレイアウト**にしてください。フェイク画像はAI生成感を加速させます。
